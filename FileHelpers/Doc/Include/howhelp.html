How to Help to the library

Development Help

Work for the library is more and more time consuming to me and I don't have full time for it.

So, if you want to help in the development please contact me at and the team will find an area of the library that you can maintain or change

You can help in any topic that you like:

  • Core Development
  • Demo App Development
  • Wizard Development
  • Write some Examples
  • MONO port
  • Help Translation (almost to english :) because I know that is full of errors)
  • Web Site Changes
  • Forums Admin
  • Or, if you dont want to work, you are welcome to donate something :P
You can change any part of the library and later you can send me your code using this excellent article as guideline:

Scott Hanselman - Example: How to contribute a patch to an Open Source Project

Another place where you can found the pending things is in the forums. We welcome all ideas and feedback to the project.

Spread to the world !!

You can vote for the article in CodeProject so that more users can know about the library and send additional feedback and ideas =)

You can also post in your blog or link a page to the FileHelpers home.

Motivation

It is always good to hear what you think about the library; in the open source world we live of the user feedback and happiness. So, send any comments to me or post them in the forums, your are always welcome.
